# Handoff: ぐるまろ！ Restaurant-matching PWA — 14 screens

## Overview

「ぐるまろ！」は、カップル・友達同士が**Tinder風スワイプ**で行きたいレストランをマッチングする、登録不要・無料の Web アプリ（PWA）です。各ユーザーが個別にスワイプし、ふたり共にYESを付けたお店が「マッチ」として表示されます。

このバンドルには、A案（ポップ振り）のモックアップ全14画面が含まれます。

- **マスコット**：ぐるまろくん（オレンジのまるい食いしん坊ゆるキャラ／5ポーズ）
- **ターゲット**：20〜30代のカップル・友達同士
- **設計方針**：アカウント登録なし／個人情報を取得しない／スマホ片手操作

## About the Design Files

このフォルダに含まれる HTML / JSX / CSS / PNG ファイルは **デザインリファレンス** です。HTML プロトタイプとして「見た目と振る舞い」を提示する目的で作られており、本番コードとして直接コピーすることは想定していません。

実装タスクは、**対象コードベースの既存環境（React / Next.js / Vue / SwiftUI / Flutter など）に、これらのデザインを再現すること**です。対象コードベースに既存の UI ライブラリ・コンポーネント・デザインシステムがある場合は、それらを使ってください。新規プロジェクトの場合は、PWA 要件に最も適したスタック（React + Vite + TypeScript + CSS Modules / Tailwind など）を選択して実装してください。

## Fidelity

**High-fidelity (hifi)**：最終的なカラー・タイポ・スペーシング・装飾を含むピクセル単位のモックアップです。配色・角丸・シャドウ・フォントウェイト・余白までこのまま再現してください。ただし、レストラン写真は SVG ストライプのプレースホルダーになっています（実装時には実画像 / Hot Pepper / 食べログ等の API レスポンスに差し替え）。

### 06 · スワイプ（YES / NO の 2状態）

スワイプの中核体験。**右スワイプ = YES（行きたい！）、左スワイプ = NO（違うかな）**（※元依頼書から方向を反転）。

- **トップバー**：ロゴ + カウンター「📍 あと10枚」（pill / `#FFE3D1`）
- **カードスタック**：
  - 背面に 2 枚プレビュー（透明度 0.7 / 0.4、scale 0.97 / 0.94）
  - 主カード：白 / 24px 角丸 / `box-shadow: 0 4px 0 rgba(61,43,31,0.08), 0 12px 24px rgba(61,43,31,0.06)`
    - 上部：レストラン写真 300px（実装時はリアル画像）
    - スタンプ：YES 状態は左上に `YES ❤`（ピンク 3px ボーダー / -12° 回転）、NO 状態は右上に `NOPE`（グレー / 14° 回転）
    - ボディ：店名（18px / 800）/ タグ群（ジャンル・エリア・予算）/ 説明文 2-3 行
- **マスコット吹き出し**（カード右下 / NO は左下）：
  - YES：like ポーズ + 「行きたいまろ〜！」（`#FF6B9A`）
  - NO：nope ポーズ + 「うーん、ちがうまろ…」（`#7B8AA0`）
- **アクションボタン**：
  - 違うかな：✕ / 64px 円 / 白 / `box-shadow: 0 4px 0 #E8DECF, 0 8px 18px rgba(61,43,31,0.12)`
  - 行きたい！：♥ / 72px 円 / グラデ `linear-gradient(180deg, #FF6B9A, #E84B7E)` / 押し込みシャドウ
  - 各ボタンの下に短いラベル（「違うかな」「行きたい！」）
- **下部リンク**：「スキップして結果を見る」（11px underline）

## Interactions & Behavior

| トリガー | 振る舞い |
|---|---|
| カード右スワイプ / ♥ タップ | YES として記録、like ポーズの ぐるまろくん が一瞬表示（300ms程度）、次のカードへ |
| カード左スワイプ / ✕ タップ | NO として記録、nope ポーズの ぐるまろくん が一瞬表示、次のカードへ |
| スワイプ中のドラッグ | カードに回転（最大 ±15°）+ オーパシティ低下 + YES/NOPE スタンプ徐々に表示 |
| 「リンクを相手に送る」タップ | URL を clipboard にコピー → 上部にトースト表示（2 秒後フェード） |
| レストラン行タップ | 行を展開し、予約 CTA を表示（高さアニメーション 200ms） |
| 予約 CTA タップ | 外部リンクを新規タブで開く（target=_blank） |
| 「結果を見る」（マッチC） | スワイプを終了し 07 / 08 へ |
| 「もう一度試す」（L3） | 失敗した API を再リクエスト |

### 状態管理（推奨）

```
type Step = 'top' | 'cuisine' | 'situation' | 'area' | 'about' | 'swipe' | 'result' | 'match'
type SwipeChoice = { restaurantId: string, choice: 'yes' | 'no' }

state = {
  step: Step,
  filters: { cuisines: string[], situations: string[], prefecture: string, areas: string[] },
  profile: { age?: string, gender?: string }, // optional
  restaurants: Restaurant[],   // 15 fetched
  swipes: SwipeChoice[],
  shareToken?: string,          // URL に埋め込む
  partnerSwipes?: SwipeChoice[] // 相手側完了後に取得
}
```

URL クエリでセッショントークンを引き継ぐ（例：`?session=abc123`）。両者のスワイプ結果はバックエンドに保存 → マッチング判定。

## 実装時のメモ

1. **PWA 化**：manifest.json + service worker。アイコンは ぐるまろくん base ベース。
2. **登録不要**：URL クエリの session token のみでセッションを引き継ぐ。
3. **PR 表記**：予約 CTA に必ず「広告を含みます」と PR タグを表示。
4. **クレジット**：フッターに Hot Pepper グルメサーチ API のクレジット表記を残す（利用規約上必須）。
5. **アクセシビリティ**：
   - スワイプはタッチに加えて、♥ / ✕ ボタンでも完全に同じ操作が可能（依頼書通り）。
   - フォーカスリングは `outline: 2px solid var(--orange); outline-offset: 2px;` を推奨。
6. **アニメーション**：依頼書記載の「スワイプ時に ぐるまろくん が一瞬表示」は約 300ms のフェードイン → 200ms 表示維持 → フェードアウトが目安。
7. **スワイプ方向**：依頼書の元記述は「左=YES、右=NO」でしたが、Tinder 等の慣習に合わせ **右=YES、左=NO** に変更済み。
